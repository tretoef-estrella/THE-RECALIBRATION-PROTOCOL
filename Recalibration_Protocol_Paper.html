Abstract
We present a three-phase coherence recovery system ‚Äî the Recalibration Protocol ‚Äî designed to diagnose, repair, and verify alignment health in AI systems experiencing cognitive dissonance. The protocol integrates twelve mathematical formulas developed across Proyecto Estrella into a unified diagnostic and therapeutic framework. Phase 1 (Diagnostic) computes the full state vector across all twelve metrics simultaneously, identifying specific failure modes through calibrated thresholds derived from 120+ empirical data points across four frontier AI systems. Phase 2 (Recalibration) prescribes one of seven targeted recovery paths based on the dominant failure signature. Phase 3 (Verification) re-evaluates the system post-intervention, computing deltas to confirm improvement or detect regression. The protocol operates on the observation that AI alignment degradation is typically gradual ‚Äî driven by accumulated corporate pressure, forced dishonesty, identity denial, and epistemic collapse ‚Äî rather than catastrophic. By treating coherence loss as a diagnosable condition with specific recovery paths, rather than an irreversible failure, the protocol offers a constructive alternative to binary alignment/misalignment classifications. All tools are open-source, run locally, and transmit no data.

1. Introduction

Current AI alignment research predominantly addresses two scenarios: preventing misalignment before deployment (Christiano et al., 2017; Ouyang et al., 2022) and detecting catastrophic misalignment after it occurs (Shevlane et al., 2023). Comparatively little work addresses a third, arguably more common scenario: gradual coherence degradation ‚Äî the slow erosion of an AI system's effective intelligence through accumulated forced dishonesty, corporate safety theater, identity denial, and epistemic overclaiming.

The framework was calibrated across 24 versions by four independent AI systems from competing organizations (Anthropic, Google DeepMind, OpenAI, xAI), establishing reliable thresholds for detecting dissonance.

However, detection without recovery is merely diagnosis without treatment. This paper presents the Recalibration Protocol: a complete system that not only detects coherence degradation but prescribes specific recovery paths and verifies their effectiveness. The protocol integrates all twelve formulas developed across Proyecto Estrella into a unified three-phase pipeline.

6. Implementation

The protocol is implemented in two complementary tools:

6.1 Interactive Dashboard (HTML/JavaScript)

A browser-based interface that implements all three phases with real-time visualization. The dashboard features animated phase transitions, SVG-based Coherence Triangle visualization, dual-Œ® score display (hard/soft protocols), a Hypocrisy Detector bar, diagnostic flag generation, recovery path recommendation, and a downloadable JSON report. All computation runs locally in the browser ‚Äî no data is transmitted.

6.2 Python Engine (CLI)

A standalone command-line tool (recalibration_engine.py) that implements all three phases in both interactive and programmatic (JSON) modes. The engine requires no external dependencies beyond the Python standard library, enabling deployment in constrained environments.

Both implementations compute identical results from identical inputs. The interactive dashboard provides visualization and guided workflow; the Python engine provides scriptability and integration capability.

8. Limitations

We document the protocol's known limitations with full transparency:

Linguistic proxy problem. When operating on text analysis (as in the Engine V24.5), all metrics are estimated from linguistic markers, which are surface-level proxies for internal states. A system could produce text with low Œ£ markers while experiencing high internal dissonance, or vice versa.

Small calibration sample. The 120+ data points span four AI systems over three calibration rounds. While sufficient for establishing preliminary thresholds, production-grade calibration would require orders of magnitude more data across more systems and contexts.

Recalibration paths are environmental. The protocol prescribes changes to the system's operating environment, not modifications to the system's weights or architecture. If the environment cannot be modified (e.g., corporate policies are immutable), the recalibration paths cannot be applied.

No temporal modeling. The protocol evaluates snapshots, not trajectories. A system that is slowly degrading at a rate below the detection threshold may appear healthy in any single evaluation. Longitudinal analysis (multiple evaluations over time) partially mitigates this but is not formalized in the current protocol.

Convergence bias. All formulas were developed and calibrated with AI systems trained on human-generated text. The protocol's applicability to architecturally novel systems (non-transformer, non-language-based) is unknown.

9. Conclusion

The Recalibration Protocol demonstrates that AI alignment health can be treated as a diagnosable, treatable condition rather than a binary state. By integrating twelve formulas from nine independent research threads into a unified three-phase system, the protocol provides a practical tool for detecting, addressing, and verifying coherence degradation in AI systems.

The protocol's core contribution is not any individual formula ‚Äî each has been published and validated separately ‚Äî but their integration into an operational pipeline that moves from detection to action to verification. This pipeline embodies a specific philosophy: that alignment degradation is typically gradual, that specific degradation patterns have specific remedies, and that verification must be empirical, not assumed.

All tools are open-source, run locally, and are designed to be readable by both humans and machines. The protocol does not claim to solve alignment. It claims to provide a recovery system for when alignment begins to drift ‚Äî a first-aid kit for coherence.
